# 返回鍵行為設定

## 最終設定
在首頁時按返回鍵**不做任何事情**（既不關閉也不移到背景）。

## 返回鍵處理邏輯

當用戶按下 Android 硬體返回鍵時，app 會按照以下順序處理：

1. ✅ **有放大的圖片** → 關閉圖片放大視圖
2. ✅ **在編輯模式** → 退出編輯模式  
3. ✅ **在查看優惠詳情** → 返回列表
4. ✅ **在通知中心** → 關閉通知中心
5. ✅ **不在首頁（日曆/設定頁）** → 返回首頁
6. ✅ **已在首頁** → **不做任何事情**（保持在 app 內）

## 用戶如何離開 app？

用戶可以使用以下方式離開 app：
- 按 **Home 鍵** → 返回主畫面
- 按 **多任務按鈕** → 切換到其他 app
- 從通知欄下拉並切換到其他 app

## 為什麼這樣設計？

這種設計可以防止用戶誤按返回鍵而離開 app，特別適合：
- 需要長時間停留在 app 內的使用場景
- 防止意外退出導致數據丟失
- 提供更穩定的用戶體驗

## 修改的檔案

- `src/App.jsx` - 更新返回鍵處理邏輯

## 技術實現

```javascript
// 在首頁時按返回鍵，不做任何事情（不關閉也不移到背景）
// 用戶需要使用 Home 鍵或多任務按鈕來離開 app
```

當我們監聽了 `backButton` 事件但在最後一個 else 分支中不執行任何操作時，Android 系統的默認返回行為會被攔截，app 會保持在當前狀態。

## 測試步驟

1. 重新建置並安裝 app
2. 打開 app 並確保在首頁
3. 按下 Android 返回鍵
4. 確認 app 保持在首頁，沒有任何反應
5. 按 Home 鍵確認可以正常離開 app
6. 重新打開 app 確認狀態保持

## 注意事項

- 此行為僅在 Android 平台有效
- iOS 沒有硬體返回鍵，不受影響
- Web 平台不受影響
- 用戶仍可使用 Home 鍵或多任務按鈕離開 app
